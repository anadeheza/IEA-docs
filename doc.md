# Chat 
## Propósito
Poder visualizar el backup de flock con una interfaz similar, que sea intuitiva, y facil de usar.

## Aprendizajes
- **Docker** -> Contenedores para apps, permite empaquetar aplicaciones y todas sus dependencias 
- **Mailhog** -> Enviar mails de pruebas ficticios con localhost en desarrollo (Puertos 1025 para envio y 8025 para visualizarlo)
- **DigitalOcean** -> se encarga del alojamiento en la nube (cloud hosting) y es proveedor de infraestructura (IaaS) para simplificar el despliegue, la gestión y el escalado de aplicaciones
- **Tarifario** -> Literalmente tarifas
- **NIC.ar** -> Para comprar dominios
- **MXToolbox** -> Para encontrar dominios
- **Prisma** -> ORM para manejar la database en Node.js  y TypeScript  
- **AdonisJS** -> Framework de backend con herramientas integradas para facilitar el desarrollo
- **Flock** -> App tipo Slack para comunicacion, con chats uno a uno y grupos/canales
- **CORS** -> Cross-Origin Resource Sharing, habilita o inhabilita a los dominios a conectar y hacer peticiones al dominio en el que estamos, principalmente se habilita al backend y al frontend a hablar entre si, con peticiones y respuestas sin problema
- **MinIO** -> almacenamiento
- **SSL/TLS** -> cifrado 
- **Opencode** -> IS en la terminal
- **Ngrok** -> Demo, sirve para hacer pruebas con servicios externos (como pagos, por ejemplo) y de uso real previos a la produccion final
- **DNS** -> Domain Name Server, traduce los nombres de dominio en direcciones IP 
- **FQDN** -> Fully Qualified Domain Name, el nombre completo del dominio, con host, dominio y TLD (Top-Level-Domain)
- **SMTP** -> Protocolo e envio de mails (no recepcion), puertos 587 (STARTTLS), 465 (SSL/TLS), 25 (sin cifrado)
- **STARTTLS** -> conexion segura mediante TLS
- **SDK** -> Software Development Kit, literalmente las herramientas necesarias para desarrollar en una plataforma o SO determinado.
- **API** -> Para conectar aplicaciones entre si con una key (clave) secreta, (generalmente tu programa con servicios externos)
